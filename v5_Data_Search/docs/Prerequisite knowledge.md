이런 방식을 슬라이딩 윈도우(sliding window)방식이라고 합니다.
읽는 크기는 8byte로 크게 보지만, 주소 사이에 걸쳐 있을지 모르는 데이터를 놓치지 않도록 1byte씩 밀어가면서 촘촘히 검사하는 것입니다.

------------------------------------------------------------------------------------------------------------------------------------------------------

왜 8byte인가.
1. 포인터(주소값)의 크기: 64bit 시스템에서 가상 메모리 주소의 길이는 8byte입니다. 그래서 메모리 속 '다른 주소'를 찾으려면 8byte 크기로 읽어야 딱 들어맞습니다.

2. 일반적인 정수(int)의 크기: C언어에서 가장 많이 사용하는 정수형인 int는 4byte입니다. 그래서 일반 숫자를 찾을 때는 4byte로 맞춥니다.

------------------------------------------------------------------------------------------------------------------------------------------------------

8byte를 뛰어넘는 거대 데이터
1. 3D 좌표 벡터 (16byte): float(4byte)가 4개(X, Y, Z, W) 연속으로 묶여있습니다.

2. 암호화 키(16byte or 32byte): AES-128이나 AES-256 같은 암호화 알고리즘에 쓰이는 키 값들은 메모리 어딘가에 연속된 바이트로 존재합니다.
   이 값은 16byte(128bit)/32byte(256bit)짜리 통짜 데이터 덩어리입니다.

3. 고유 시그니처 / 기계어 패턴: 안티바이러스가 악성코드를 탐지하거나, 해커가 업데이트되어도 변하지 않는 특정 함수를 찾을 때 쓰는 '기계어'의 흐름입니다.
   특정 함수 구조를 통째로 매칭해야 하므로 20byte, 32byte, 혹은 그 이상의 크기를 가집니다.

4. 구조체 및 오브젝트: 플레이어의 이름, 등급, 스탯처럼 묶여있는 오브젝트 덩어리는 메모리에 수백 byte 크기로 존재합니다. 

------------------------------------------------------------------------------------------------------------------------------------------------------

8byte가 넘는 데이터는 어떻게 검색하는가?
*(int)나 *(uintptr_t*) 같은 포인터 형변환 비교 공식을 사용할 수 없습니다.
CPU 레지스터에 16byte짜리 숫자를 한 번에 올려서 ==기호로 비교하는 하드웨어 명령어가 일반적인 연산에서는 없습니다.

그래서 8byte보다 큰 데이터를 찾을 때는 배열 대조 방식을 사용합니다.

unsigned char target_key[16] = { 0xAA, 0xBB, 0xCC, ...... };
if (memcmp(base_addr, target_key, 16) == 0){
    ...
}
같이 memcmp 등을 사용하여 배열 대조를 합니다.

------------------------------------------------------------------------------------------------------------------------------------------------------

memcmp? 
<string.h>에서 제공하는 함수입니다.
지정한 데이터와 동일할 때만 0을 반환합니다.

예시의 코드에 target_key[16]은 왜 16이냐? 예상하시다 시피 16byte를 예시로 들었기 때문입니다.
만약 4byte 데이터를 찾는다면 target_key[4]였을 것입니다.

memcmp(base_addr, target_key, 16) == 0
부분도 동일하게 16byte만큼만 대조하겠다는 의미입니다.
4byte 데이터를 찾는다면 4로 수정하면 됩니다.

target_key에 있는 {0xAA, 0xBB, 0xCC...}는 뭐냐?
찾고자하는 데이터를 1byte씩 순서대로 나누어서 넣습니다.
