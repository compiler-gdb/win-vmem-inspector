메모리 주소 변환: addr_min, addr_max 및 next_area 변수를 포인터로 선언하여 메모리 주소를 직접 조작합니다. 
그러나 코드에서는 이러한 변수를 직접 더하거나 빼고 있습니다. 이는 C 언어에서 올바른 방법이 아닙니다. 
포인터 연산을 사용할 때 주의해야 합니다.


본인: Min_addr+=1 처럼 작성했었음.
addr += 1;은 주소에 1을 더하는 것입니다. 이는 주소가 가리키는 실제 메모리 위치를 변경합니다. 이 연산을 사용하면 현재 위치에서 1바이트 앞으로 이동하게 됩니다.
++addr, addr++은 포인터를 증가시키는 연산입니다. 
++addr - 주소를 하나 늘려서 다음 메모리 위치를 가리키게 합니다. 
addr++ - 현재 포인터가 가리키는 메모리 주소에 값을 가져오고 포인터를 한 바이트만큼 더 나아가게 함.

포맷 지정자: printf 함수에서 %s와 같은 올바른 포맷 지정자를 사용해야 합니다. 현재 코드에서는 printf(mbi.State)가 문자열을 출력하는 대신 메모리 주소를 해석할 수 있습니다.
