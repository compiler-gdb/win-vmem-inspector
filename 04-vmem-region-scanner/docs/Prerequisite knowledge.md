v3의 연장선입니다.
밑의 3가지 기능 추가하는걸 목표로 합니다.

1. 데이터 보기 (Inner Scanning Loop)
VirtualQuery로 찾은 구역(Region) 내부를 처음부터 끝까지 검사합니다.

2. 권한 확인 (Permission Guard)
모든 MEM_COMMIT 영역을 다 읽으려 하면 프로그램이 죽을 수 있습니다. 권한이 없는 데이터가 있을 수 있기 때문입니다.
추가 로직: mbi.Protect를 확인해서 읽기 권한이 확실히 있을 때만 진입합니다.

v3_Memory_State에서는 "어떤 상태"인지만 보고 "데이터"에는 접근하지 않아서 권한에 영향을 받지 않았습니다.


3. 타겟 매칭 (Value Comparison)
내가 찾고 싶은 값(예: 특정 게임의 점수, 비밀번호 조각 등)과 일치하는 주소를 찾아냅니다.

state(메모리 상태)
1. MEM_COMMIT 상태인지 MEM_FREE 상태인지 MEM_RESERVE 상태인지 확인합니다.

2. MEM_RESERVE는 메모리 할당은 되었지만 가상 메모리에서 어디를 사용할지 예약하는 것입니다.

    2-1)읽기/쓰기는 불가능합니다.

    2-2)왜 쓰냐? 1GB 가변형 배열을 만든다 가정하고 처음부터 싹 다 COMMIT을 하면 자원 낭비가 되고, 1GB만큼의 공간을 가상에서 예약합니다. "1~50째 칸까지 비워줘" 같이 말입니다. 다 같이 모여 앉을 수 있게 됩니다.
    
    2-3)파편화: 예약없이 필요할 때마다 조금씩 할당하면 빈 자리 아무 곳에나 데이터나 프로그램이 할당됩니다. 총 30개 간이 남았다 가정하면 20개 칸이 필요해도 연속적이지 않아서 30개의 칸이 있을지라도 할당하지 못하게 되는 경우가 생깁니다.
